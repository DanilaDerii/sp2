SP2 Roadmap - Current
=====================

Last audited: 2026-06-30


Done
----
1. Teacher PDF pack pipeline.
2. Teacher pack export as `pack.json`, `chunks.json`, `vectors.npy`, and `.zip`.
3. Student SQLite installed-pack storage.
4. Student LanceDB `pack_chunks` storage.
5. Student direct `.zip` pack importer.
6. Student retrieval modules.
7. Student FastAPI routes for packs and retrieval context.
8. LM Studio MCP server and tools.
9. End-to-end LM Studio -> SP2 -> LM Studio retrieval path proved.


Current Priority
----------------
Improve answer quality now that the main business logic is in place.

Direction guardrails:
- Treat SP2 as one app/install with teacher and student modes.
- Keep the internal `teacher/` and `student/` code split for module clarity.
- Move embedding-provider direction from Ollama to LM Studio local embeddings
  API for both pack creation and query retrieval.
- LM Studio should become the local model provider for chat and embeddings.

1. Improve retrieval ranking.
2. Add adjacent chunk expansion or context windows.
3. Improve teacher chunking so chunks contain better concepts.
4. Switch and test stronger LM Studio-served local embedding models.
5. Consider hybrid keyword + vector retrieval.
6. Consider reranking after initial LanceDB retrieval.
7. Tune MCP tool descriptions and returned packet guidance for LM Studio.
