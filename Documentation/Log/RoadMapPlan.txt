Current Priority
----------------
Improve answer quality now that the main business logic is in place.

Direction guardrails:
- Treat SP2 as one app/install with teacher and student modes.
- Keep the internal `teacher/` and `student/` code split for module clarity.
- LM Studio is the local model provider for chat and embeddings.
- Keep `text-embedding-nomic-embed-text-v1.5` as the single v1 embedding model
  for pack creation and query retrieval.

1. Improve retrieval ranking.

3. Improve teacher chunking so chunks contain better concepts.

5. Consider hybrid keyword + vector retrieval.

Next Product Work

Pack versioning direction for future implementation:
- If a teacher imports a file with the same logical pack name but changed
  content, handle it as a version bump instead of silently replacing the old
  install.
- Example:
  `week01 v1`
  `week01 v2`
- Retrieval should continue to use the local `installed_pack_id`; `pack_id` plus
  `version` identifies the logical course-pack version.
- Do not overwrite an installed pack in place until explicit replacement
  semantics are designed.

Pack deletion direction for future implementation:
- Current code has low-level delete helpers for SQLite installed-pack rows and
  LanceDB chunk rows, but no complete public uninstall workflow.
- Add a student-side pack management service before exposing deletion through
  API or MCP.
- A complete uninstall should remove:
  `installed_packs` SQLite row
  `pack_chunks` LanceDB rows
  installed pack files under `student/storage/installed_packs/`
- Expose deletion after the service exists:
  `DELETE /packs/{installed_pack_id}`
  `sp2_delete_pack(installed_pack_id)`
- Retrieval should continue to reject inactive or deleted packs by
  `installed_pack_id`.


Later Product Work
------------------

Implementation guardrails for this later work:
- Keep one SP2 MCP server exposed to LM Studio.
- Keep MCP thin; put build/import orchestration behind backend/service code.
- Reuse the existing teacher pack builder, student importer, and retrieval
  modules instead of merging their responsibilities.
- Keep LM Studio as the chat UI and final-answer owner.
